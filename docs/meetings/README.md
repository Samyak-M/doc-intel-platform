# Meeting Notes & Async Updates

This folder contains all meeting summaries, action items, and async update logs.

## How to Document a Meeting

After each working session, create a file with this format:

**File name:** `YYYY-MM-DD_meeting_summary.md`  
**Example:** `2026-06-22_kickoff_meeting.md`  
**Deadline:** Within 24 hours of meeting

### Template

```markdown
# Meeting Summary: [Title]

**Date:** YYYY-MM-DD  
**Time:** HH:MM - HH:MM [Timezone]  
**Attendees:** [List]  
**Owner/Facilitator:** [Name]  

## Agenda

1. Topic 1
2. Topic 2
3. Topic 3

## Decisions Made

- **Decision 1:** [Recommendation] → [Who decided]
- **Decision 2:** [Recommendation] → [Who decided]

## Action Items

| Owner | Task | Due Date | Status |
|-------|------|----------|--------|
| Name | Description | YYYY-MM-DD | Pending / In Progress / Complete |

## Blockers

- Blocker 1 (Owner: Name)
- Blocker 2 (Owner: Name)

## Next Steps

- [ ] Action item 1
- [ ] Action item 2

## Related Issues

[Link to relevant GitHub issues tracking follow-up work]

---

**Next Meeting:** [Date/Time if scheduled]
```

## Async Updates

If you can't attend a meeting, post an async update:

**File name:** `YYYY-MM-DD_async_update_[yourname].md`

```markdown
# Async Update: [Your Name]

**Date:** YYYY-MM-DD  
**Period:** [Week or work period]  

## What I Worked On

- Task 1 — Status
- Task 2 — Status
- Task 3 — Status

## Progress

Brief description of progress this week.

## Blockers

Any blockers preventing progress?

## Next Week

What will you work on next?

## Questions or Notes

Any questions for the team or notes to share?
```
